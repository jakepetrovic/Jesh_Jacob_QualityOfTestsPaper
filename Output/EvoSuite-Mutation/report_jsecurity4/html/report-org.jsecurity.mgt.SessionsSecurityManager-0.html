 39:  * JSecurity support of a {@link SecurityManager} class hierarchy that delegates all
 40:  * {@link org.jsecurity.session.Session session} operations to a wrapped {@link SessionManager SessionManager}
 41:  * instance.  That is, this class implements the methods in the
 42:  * {@link SessionManager SessionManager} interface, but in reality, those methods are merely passthrough calls to
 43:  * the underlying 'real' <tt>SessionManager</tt> instance.
 44:  *
 45:  * <p>The remaining <tt>SecurityManager</tt> methods not implemented by this class or its parents are left to be
 46:  * implemented by subclasses.
 47:  *
 48:  * <p>In keeping with the other classes in this hierarchy and JSecurity's desire to minimize configuration whenever
 49:  * possible, suitable default instances for all dependencies will be created upon instantiation.
 50:  *
 51:  * @author Les Hazlewood
 52:  * @since 0.9
