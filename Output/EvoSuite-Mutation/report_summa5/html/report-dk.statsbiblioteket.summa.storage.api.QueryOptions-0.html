 24: /**
 25:  * {@code QueryFilter} is an immutable filter that is passed into the methods
 26:  * in the {@link ReadableStorage} interface to add additional criteria the
 27:  * records must match.
 28:  * <p/>
 29:  * There are four core options one can set on a query. It is possible to add
 30:  * additional custom options to the storage by passing them in via the
 31:  * {@code meta} argument.
 32:  * <p/>
 33:  * The main options are:
 34:  * <ul>
 35:  *   <li>{@code deletedFilter} : If {@code null} allow all records disregarding
 36:  *                              their state. If {@code true} match only deleted
 37:  *                              records, and if {@code false} match only records
 38:  *                              that are not deleted</li>
 39:  *   <li>{@code indexableFilter} : If {@code null} allow all records disregarding
 40:  *                             their state. If {@code false} match only records
 41:  *                             not indexable, and if {@code true} match only
 42:  *                             records that are marked indexable</li>
 43:  *   <li>{@code childDepth} : The number of child levels to expand children to.
 44:  *                            If this value is {@code 0} then no expansion will
 45:  *                            occur, if it is {@code -1} expansion will occur
 46:  *                            recursively to any depth</li>
 47:  *   <li>{@code parentHeight} : Number of levels to expand parent records
 48:  *                              upwards. If this value is {@code 0} then no
 49:  *                              parent expansion occurs, and if it is
 50:  *                             {@code -1} do recursive expansion up to the root
 51:  *                             record</li>
 52:  * </ul>
 53:  *
 54:  * <h3>Null Options</h3>
 55:  * If the whole {@code QueryOptions} object or any one of the options on it
 56:  * is {@code null} it indicates a fully permisive option.
 57:  * A {@code QueryOptions} set to {@code null} will allow all records and do no
 58:  * expansion of children or parents.
 59:  *
 60:  * <h3>Meta flags</h3>
 61:  * Query options can host an arbitrary map of key/value pairs known as meta
 62:  * flags. These flags can be used to alter the behaviour of queries a to pass
 63:  * sepcial information to batch jobs.
 64:  * <p/>
 65:  * Pre defined meta flags:
 66:  * <ul>
 67:  *   <li><tt>ALLOW_PRIVATE</tt> - Set this to {@code "true"} to allow
 68:  *       {@code getRecord()} requests to access storage private records,
 69:  *       like {@code __holdings__}</li>
 70:  *   <li><tt>TRY_UPDATE</tt> - If this flag is set this to {@code "true"} any
 71:  *       calls to {@code flush()} or {@code flushAll()} will not update
 72:  *       the record if it already exists with the exact same fields. This also
 73:  *       means that the record modication time will not be updated if it is
 74:  *       already known</li>
 75:  * </ul>
 76:  */
